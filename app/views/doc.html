Proyecto

Es mi primera vez con mongoDB. Hice CRUD básico para las hamburguesas y las papas fritas.
El menu se puede crear y listar con la relación de hamburguesa y papas.
He cambiado la estructura ya que solo usar el index y modelos quedaría muy espagueti. Preferí un modelo MVC.
En los modelos estan los body para los endpoint, igual la db tiene todo para las pruebas.

Recursos disponibles:

Menus

GET /menus → lista el menu con relaciones

POST /menus → crear menu

Ejemplo: 
            
                {
                    "name": "Combo Extra Mega Big Bacon Grande",
                    "price": "1100",
                    "burger": "634cbd2de5740433f562325c",
                    "fries": "634cb6d22778e13c5829cf95"
                }

Burgers

GET /burgers → lista todas las burgers

POST /burgers → crear burger

Ejemplo: 
            
                {
                    "name": "La clasica que nunca falla",
                    "price": "800",
                    "ingredients": {
                        "bread": true,
                        "burgers": 2,
                        "lettuce": true,
                        "tomato": true,
                        "cheese": true,
                        "bacon": false,
                        "sauce": "Mayo"
                    }
                }

GET /burgers/:key/:value → obtener datos de burguer segun parametros de busqueda o lista todos

Ejemplo: 
            
                GET /burgers/_id/634caa95e57d739001be6fda

PUT /burgers/:key/:value → actualizar burger

DELETE /burgers/:key/:value → borra burger

Papas Fritas

GET /fries → lista todas las burgers

POST /fries → crear burger

Ejemplo: 
            
                {
                    "name": "Papas Chicas",
                    "price": "50",
                    "size": "Small"
                }

GET /fries/:key/:value → obtener datos de papas fritas segun parametros de busqueda o lista todos

PUT /fries/:key/:value → actualizar papas

DELETE /fries/:key/:value → borra papas
